What are Data URIs?

Data URIs (also called data URLs) let you embed file content directly in HTML, CSS, or JavaScript as a string, without needing a separate HTTP request. The format is:

data:[mediatype][;base64],data

For example, a small red dot PNG:

data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mP8/5+hHgAHggJ/PchI7wAAAABJRU5ErkJggg==

Common Use Cases

  • CSS backgrounds — embed small icons or patterns directly in stylesheets
  • Inline images in HTML — reduce HTTP requests for small images
  • SVG embedding — embed SVG icons without extra files
  • Favicons — embed favicons directly in HTML <link> tags
  • Email HTML — embed images in email templates (some clients support this)

Size Recommendations

  • IE 8 — 32 KB limit (no longer relevant for most projects)
  • Modern browsers — practical limit around 2 MB, though there is no hard spec limit
  • Performance — data URIs increase CSS/HTML file size by ~33% (base64 overhead). For files over 10-20 KB, a separate file with HTTP caching is usually faster
  • Best for — small images (<5 KB), SVG icons, fonts used in critical CSS

Base64 vs Plain Text

Binary files (images, fonts) must use base64 encoding. Text-based formats (SVG, HTML, CSS, JSON) can be embedded without base64 — just URL-encode special characters. Skipping base64 for text content produces smaller data URIs.
